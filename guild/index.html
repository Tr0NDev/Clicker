⚙️ Developer Setup — Supabase

This guide is for whoever hosts the game. Players don't need to do anything — the connection is shared.

📋 Step-by-step Supabase setup ▼
  • 1
    Go to supabase.com, create a free account and click New Project. Pick a name (e.g. "clicker-game"), choose a region close to your players, set a database password.
  • 2
    In your project dashboard, go to Project Settings → API. Copy the Project URL and the anon / public key.
  • 3
    Go to SQL Editor (left sidebar) and run the following query to create the guilds table:
    create table guilds ( id uuid default gen_random_uuid() primary key, name text not null unique, owner_id text not null, member_count int not null default 1, members jsonb not null default '[]', created_at timestamptz default now() ); -- Allow anyone to read guilds create policy "Read guilds" on guilds for select using (true); -- Allow authenticated/anon users to insert create policy "Create guild" on guilds for insert with check (true); -- Allow owner to update their guild create policy "Update own guild" on guilds for update using (owner_id = current_setting('request.jwt.claims', true)::json->>'sub' or true); -- Allow owner to delete their guild create policy "Delete own guild" on guilds for delete using (true); alter table guilds enable row level security;
  • 4
    Back in your guild page source code, find these two lines at the top of the <script> and replace the placeholder values:
    const SUPABASE_URL = "https://YOUR_PROJECT.supabase.co"; const SUPABASE_ANON_KEY = "eyJ...";
  • 5
    Deploy the updated file. All players who open the page will automatically connect to the same shared database. No login required — each player gets a persistent anonymous ID stored in their browser.
  • 6
    Optional security: In Supabase → Authentication → Settings, you can leave anonymous sign-ins off. The anon key already scopes access to the RLS policies you set up in step 3. For a more secure setup, enable Supabase Auth anonymous sign-ins and remove the or true from the update/delete policies.
